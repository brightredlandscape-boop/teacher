Introduction1.

1 PurposeThis Software Requirements Specification (SRS) defines the complete functional and non-functional requirements for TeachLink Africa.

It is the authoritative reference document for product, engineering, design, and QA teams throughout all development phases.

1.

2 Project OverviewTeachLink Africa is a premium two-sided marketplace connecting verified African teachers with local and international parents and students for private and home tutoring.

The platform operates across web and mobile, supports multi-currency transactions, integrates live session infrastructure with time-clocking, and includes an AI-powered teacher upskilling academy.

It competes directly with Preply, Superprof, AmazingTalker, TeacherOn, and Cambly.

5 Scalability and InternationalisationFirestore horizontal scaling supports growth without architectural changes.

All media assets served via CDN (Cloudinary CDN / Firebase CDN) for global latency reduction.

Firebase Cloud Functions process all webhooks, payouts, and scheduled jobs asynchronously.

Backend architecture supports React Native mobile app (Phase 6) without changes.

Gemini API usage shall be rate-limited and batched to stay within Google AI Studio quotas.

Default language: English.

Planned: French (Phase 6), Swahili, Yoruba, Hausa (later phases).

All timestamps stored in UTC; displayed in the user&apos;s detected local timezone.

Platform shall support RTL text rendering for future Arabic language support.
